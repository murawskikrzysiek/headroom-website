Audita · Automation API

Read the room,
from anywhere.

Audita exposes live SPL, dose, and session state through an audita:// URL scheme with x-success callbacks, plus session and monitoring controls. The same actions are available as App Intents in Shortcuts.

v1.2·App Intents require macOS 14+

Overview #

The audita:// URL scheme lets any tool that can fire a URL read Audita's live measurements or control sessions — Shortcuts, Stream Deck, Raycast, AppleScript, or open from a shell. Read endpoints return data through an x-success callback; control endpoints are fire-and-forget (with optional x-success/x-error).

# Read current dose into a Raycast script
open "audita://dose?x-success=raycast://run?id=show-dose"

# Start a session
open "audita://session/start?name=Mix%20review"

Read endpoints #

READaudita://spl?x-success=…

Returns the current sound level and short-term loudness.

Returns
dba, dbc (2dp); leq_1m, leq_10m, leq_30m
x-error
no_input_device
READaudita://dose?x-success=…

Returns today's accumulated hearing dose.

Returns
dose_percent (0–∞), tier (safe|caution|warning|over), lex_today, lex_week
READaudita://session?x-success=…

Returns the active session, or active=false.

Returns
active, name, duration_s, avg_dba, peak_dba, dose_contribution
READaudita://state?x-success=…

One round-trip: everything the three reads above return, plus on_target_percent and input_device.

Returns
All SPL + dose + session fields, on_target_percent, input_device

Session & monitoring #

WRITEaudita://session/start?name={text}

Start a session manually. name is optional.

x-error
session_already_active, no_input_device
WRITEaudita://session/end

End the active session and generate its PDF report.

x-success
Appends duration_s, dose_contribution, result=ok
x-error
no_active_session
WRITEaudita://monitoring/{pause|resume}

Pause or resume dose accumulation — e.g. to exclude a loud non-work interruption.

x-error
monitoring_unavailable

Shortcuts intents #

The same actions are exposed as App Intents under Audita in the Shortcuts app (macOS 14+), each returning structured values:

IntentReturns / Parameters
Get Current SPLdBA, dBC, Leq
Get Hearing Dosepercent, tier, today's L_EX
Get Sessionactive, name, duration, avg/peak
Start / End Sessionname (optional)
Pause / Resume Monitoring—

Error codes #

CodeMeaning
no_input_deviceNo measurement input is selected or available.
monitoring_unavailableMonitoring can't run — missing mic permission or no input.
session_already_activeA session is already running; end it first.
no_active_sessionNo session to end or query.
invalid_argumentA query parameter was malformed.
unknown_endpointURL path didn't match. Only fires when x-error was supplied.

Examples #

Stream Deck — start/end a session

Button "Session ▶":  System » Open → audita://session/start?name=Tracking
Button "Session ■":  System » Open → audita://session/end

Shortcuts — warn me when dose gets high

1. Open URLs → audita://dose?x-success=shortcuts://run-shortcut?name=DoseCheck
--- "DoseCheck":
2. Get value for "tier" in Shortcut Input
3. If "tier" is "warning" or "over" → Show Notification "Ears: take a break"
