Lyra · Automation API

Two surfaces.
One state.

Lyra exposes two scriptable surfaces: a bundle of App Intents for Shortcuts, Siri, and Focus Filters, and a lyra:// URL scheme for everything else. Both map onto the same underlying state.

Two surfaces, same state #

App Intents are the native macOS Shortcuts surface. Drag them into a Shortcut, speak them to Siri via a named Shortcut, or wire them into a Focus Filter. They get typed parameters, structured return values, and proper error dialogs. Require macOS 13 (Ventura) or later.

The lyra:// URL scheme is the lowest-common-denominator surface for everything else: Stream Deck, Keyboard Maestro, Raycast, AppleScript, open from a shell. Action URLs are fire-and-forget; read URLs (new in v1.5) return live state via x-success callbacks.

open "lyra://reference/2"
open "lyra://volume/set?tapered=0.4"
open "lyra://mute/toggle"

# Read with callback
open "lyra://state?x-success=raycast://run?id=announce"

Unknown URLs are logged and silently ignored. Action URLs require Lyra running and your Apollo reachable; otherwise Lyra fires x-error with mixer_disconnected.

v1.5

The eleven intents #

All eleven intents live under Lyra in the Shortcuts action library, gated on macOS 13+. Every intent returns structured values — use Get Property of <Lyra State> to extract fields. No URL parsing, no JSON decode.

State (read)

IntentReturns
Get Lyra StateTapered, dB, muted, dim, mono, mixer/device connected, device name, active slot, volume step
Get Current Monitor LevelTapered (0–1) and dB
Get Current Reference LevelActive slot (1–3) + name + tapered + dBSPL, or a "no active slot" sentinel

Monitor (write)

IntentParameters
Set Monitor LevelPercent (0–100). Clamped.
Adjust Monitor LevelDirection (up/down), Fine (half-step)
Toggle Mute / DIM / Mono(no parameters — three separate intents)

Reference Levels (write)

IntentParameters
Switch to Reference LevelSlot (1–3) or Name (case-insensitive)
Cycle Reference Level(no parameters)
Save Current Level to Reference SlotSlot (1–3)
v1.5

x-callback-url #

Two optional query parameters on every endpoint:

ParamFires when
x-successThe endpoint completed. Returned values are appended as query items, plus result=ok for action endpoints.
x-errorThe endpoint failed. Lyra appends error_code and error_message.

Example — action chain

# Set volume to 0.4, then trigger a Raycast script
open "lyra://volume/set?tapered=0.4&x-success=raycast://run?id=announce-mix"

# What Raycast receives after the set completes:
raycast://run?id=announce-mix&db=-12.34&result=ok&tapered=0.4000

Volume endpoints #

WRITElyra://volume/set?tapered={0.0..1.0}

Set the Apollo monitor level to an absolute tapered value. Same scale as UA Mixer Engine's CRMonitorLevelTapered.

Params
tapered — float, 0.0–1.0, required. Out-of-range values clamped.
x-success
Appends tapered, db, result=ok
x-error
invalid_argument, mixer_disconnected, action_failed
WRITElyra://volume/up[?fine=1]

Step the monitor level up by one increment (default 1/16 of full range). fine=1 halves the step — equivalent to holding Shift with F12.

x-success
Appends tapered, db, result=ok
x-error
mixer_disconnected, action_failed
WRITElyra://volume/down[?fine=1]

Mirror of volume/up. Same fine=1 flag for half steps.

x-error
mixer_disconnected, action_failed
READv1.5lyra://volume/get?x-success=…

Returns the current monitor level via the supplied x-success URL. Without x-success it logs the request and does nothing.

Returns
tapered (float, 4dp, 0–1); db (float, 2dp)
x-error
mixer_disconnected

Toggle endpoints #

WRITElyra://{mute|dim|mono}/toggle

Flip mute, DIM, or mono. Three endpoints, identical shape.

x-success
Appends the new boolean state (e.g. muted=true), result=ok
x-error
mixer_disconnected, action_failed

Reference Level endpoints #

WRITElyra://reference/{1|2|3|cycle}

Jump to slot 1–3, or cycle to advance to the next slot.

x-success
Appends slot, slot_name, tapered, db, result=ok
x-error
invalid_slot, mixer_disconnected
WRITElyra://reference/{n}/save?name=Mix&dBSPL=85

Write a slot. Lyra reads the live Apollo level for the tapered value; name and dBSPL are optional. This is the endpoint Audita uses for "Send to Lyra…".

Params
name — string, optional · dBSPL — number, optional
x-error
invalid_slot, invalid_argument
READv1.5lyra://reference/current?x-success=…

Returns the active slot, or slot=none when the live level matches no slot. Always includes live tapered and db.

Returns
slot, slot_name, slot_tapered, slot_dbspl, tapered, db
v1.5

Full state endpoint #

READv1.5lyra://state?x-success=…

Returns the full app state in one round-trip — everything the volume and reference reads return, plus the toggles, device info, and step size. Ideal for populating a Shortcut, Stream Deck overlay, or status dashboard.

Returns
tapered, db — current level
muted, dim, mono — bools
mixer_connected, device_connected — bools
device_name — e.g. "Apollo Twin X"
volume_step — float, typically 0.0625
slot, slot_name, slot_tapered, slot_dbspl — active slot, or slot=none
x-error
Rare — succeeds even if the mixer is disconnected (returns mixer_connected=false).

Error codes #

When an endpoint fails and an x-error URL was supplied, Lyra opens it with error_code and error_message appended. Codes are stable strings — branch on these, not the message text.

CodeMeaning
mixer_disconnectedUA Mixer Engine isn't reachable. Try Start UA Mixer Engine from Lyra's menu.
device_not_foundNo compatible Apollo present on the system.
invalid_slotThe slot number in the path was not 1, 2, or 3.
invalid_argumentA query parameter was malformed — missing key, out-of-range, or non-numeric.
mixer_send_failedTCP write failed mid-flight. Usually transient — retry.
action_failedCatch-all for an action that completed without throwing but didn't change state.
unknown_endpointURL path didn't match. Only fires when x-error was supplied; otherwise logged and ignored.

Integration examples #

Stream Deck — three K-System buttons

Three buttons, each calling a Reference Level slot. Use the built-in System » Open action — no plugin needed.

Button 1 (K-14):   System » Open → lyra://reference/1
Button 2 (K-20):   System » Open → lyra://reference/2
Button 3 (K-Free): System » Open → lyra://reference/3

Apple Shortcuts — read state, post to Slack

Shortcut: "Post current mix to Slack"
1. Open URLs → lyra://state?x-success=shortcuts://run-shortcut?name=PostMix-Continue

--- Second shortcut, "PostMix-Continue":
1. Get value for "db" in Shortcut Input
2. Get value for "slot_name" in Shortcut Input
3. Text: "Mixing at [db] dB (slot: [slot_name])"
4. Get Contents of URL → POST https://hooks.slack.com/... with the text
