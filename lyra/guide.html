§ 1

Quick start #

  1. Connect your Apollo and make sure UA Console (or just UA Mixer Engine) is installed.
  2. Launch Lyra. Grant Accessibility permission when prompted — Lyra needs it to intercept F10 / F11 / F12 globally.
  3. Once the menu shows "Connected: <your Apollo>", press F12 / F11 / F10 to raise / lower / mute monitor volume.
  4. Add modifiers for more: ⇧ for fine steps, ⇧F10 for DIM, ⌥F10 for Mono, ⌃F10/⌃F11/⌃F12 to jump to a saved Reference Level.

That's the whole app. Everything below is detail.

§ 2

First launch & Accessibility permission #

Lyra intercepts F10 / F11 / F12 globally using a CGEventTap. This is the same mechanism macOS itself uses to handle media keys, and it requires Accessibility permission.

On first launch:

  1. Lyra shows the macOS system Accessibility dialog. Click Open System Settings.
  2. In System Settings → Privacy & Security → Accessibility, find Lyra and toggle it on.
  3. If Lyra isn't in the list yet, click + and add it from /Applications.
  4. Relaunch Lyra after enabling the toggle.

Resetting Accessibility for testing

If the system gets into a weird state after reinstalls, clear the TCC entry:

tccutil reset Accessibility com.headroom.Lyra

What about all my keystrokes?

Lyra only reads three keys (NX_KEYTYPE_MUTE, NX_KEYTYPE_SOUND_DOWN, NX_KEYTYPE_SOUND_UP) — the system-defined media-key events. It does not log, store, or forward anything.

§ 3

Keyboard shortcuts #

All shortcuts work globally and are intercepted before any other app sees them — but only when Mixer Engine is reachable. When it isn't, the F-keys fall through to whatever the system would otherwise do.

ShortcutAction
F12Volume up — one step
F11Volume down — one step
F10Mute / Unmute
⇧F12Volume up — fine step (½)
⇧F11Volume down — fine step (½)
⇧F10DIM — toggle Apollo hardware DIM
⌥F10Mono — toggle L+R sum to mono
⌃F10Jump to Reference Level slot 1
⌃F11Jump to Reference Level slot 2
⌃F12Jump to Reference Level slot 3

About the step size

Lyra uses a default step of 1/16 of full range (~6.25%). Shift halves it to ~3.125% — the same pattern macOS uses for built-in volume keys. The step size is adjustable in Settings.

§ 4

Volume HUD #

Whenever you change the monitor level, Lyra shows a subtle on-screen pill — the same idea as the macOS built-in volume overlay. It reflects the current level, turns amber when DIM is active, and when you jump to a Reference Level it labels itself with the slot name (e.g. Mix, or Mix — 85 dB SPL when calibrated).

Prefer no overlay? Turn off Show volume HUD overlay in Settings.

§ 5

Reference Levels #

Lyra has three monitor presets, each stored as a tapered position (0.0–1.0) plus a user-renamable name. Defaults are Quiet (25%), Mix (50%), Loud (75%).

Jumping to a slot

  • Keyboard: ⌃F10 → slot 1 · ⌃F11 → slot 2 · ⌃F12 → slot 3.
  • Menu: Reference Levels submenu → click the slot's name.
  • URL: open lyra://reference/2 (slots are 1-indexed in URLs).

Jumping is a one-shot — Lyra writes the stored tapered value straight to the Apollo's CRMonitorLevelTapered. No ramping, no smoothing.

ⓘ

Calibration data is cleared on overwrite. If a slot had a measured dB SPL value attached (from Audita), manually saving over it clears the SPL value, because the prior measurement was tied to the old level.

§ 6

Audita pairing #

Audita, our SPL meter, can anchor a Lyra slot to a measured loudness. Run Audita's target-level calibration, then click "Send to Lyra…" — Audita ships the calibrated dB SPL to Lyra via lyra://reference/{n}/save, and the slot starts displaying as e.g. "Mix — 85 dB SPL" in the menu and HUD. Requires Audita 1.0 or later.

§ 7 v1.6

Widgets #

Lyra ships four interactive widgets for the macOS widget gallery (Notification Center or the desktop). Each shows live state — Mute and Mono tiles highlight when active, the current reference slot highlights, and buttons dim when Lyra isn't connected — and reacts the instant you tap. Every tap fires a lyra:// URL so the action runs in the menu bar app.

WidgetContents
Control Panel (large)Volume, Mute / DIM / Mono, and the three reference slots in one tile
Monitor Toggles (medium)Mute / DIM / Mono
Volume (small/medium)Step the monitor level up and down
Reference Levels (medium)Jump to slot 1, 2, or 3
ⓘ

Widgets require macOS 14 or later. On macOS 12 and 13 Lyra runs identically without the widget gallery entries.

§ 8 v1.6

Settings & automations #

Open Lyra's Settings window with Cmd+,. It holds the volume HUD toggle, the volume step size, and two opt-in automations (both off by default), each with its own notification sub-toggle:

  • Auto-start UA Mixer Engine when Apollo is detected — Lyra opens UA's headless Mixer Engine the moment Apollo wakes up.
  • Auto-quit UA apps when Apollo disconnects — after ~7 seconds without Apollo, Lyra quits UA Mixer Helper, UA Mixer Engine, UAD Console, and UAD Meter & Control Panel so they stop drawing power. Cancelled if Apollo returns inside the grace window, so brief sleep/wake won't kill anything.
§ 9 v1.5

URL scheme (lyra://) #

Lyra registers a custom lyra:// URL scheme so any tool that can fire a URL can drive it — Stream Deck, Keyboard Maestro, Apple Shortcuts, BetterTouchTool, AppleScript, or other apps. The API reference documents every endpoint; the essentials:

open "lyra://reference/2"
open "lyra://volume/set?tapered=0.4"
open "lyra://volume/up?fine=1"
open "lyra://mute/toggle"

As of 1.5, read endpoints (lyra://volume/get, lyra://reference/current, lyra://state) return live state via an x-success callback, and every endpoint accepts x-success/x-error for chaining. See the API reference for the full contract.

§ 10 v1.5

Shortcuts, Siri & Focus #

Lyra exposes eleven App Intents. They appear in the Shortcuts app under Lyra, surface in Spotlight as pre-wired App Shortcuts, can be spoken to Siri via a named Shortcut, and plug into Focus Filters.

Focus Filter

In System Settings → Focus → Focus Filters, add Lyra's "Switch to Reference Level" filter to any Focus. Lyra jumps to that slot when the Focus turns on, with an optional "Restore previous level when Focus ends" toggle. The common setup: a Mixing Focus that jumps to slot 2 and silences non-DAW notifications.

Requires macOS 13+. On macOS 12, Lyra runs identically and this surface is simply hidden.

§ 11

UA Mixer Engine #

Lyra talks to UA Mixer Engine — the background process installed with UA Console — over a local TCP connection (port 4710). It reads and writes the monitor level directly; Console is never required to be open. Nothing leaves your machine.

If Lyra shows "UA Mixer not running", your Apollo is detected but the engine isn't up. Use Start UA Mixer Engine from Lyra's menu (or enable the auto-start automation in Settings).

Protocol reverse-engineering credit: Radu Varga.

§ 12

Trial & licensing #

Lyra is a 7-day free trial, then a one-time $9.99 purchase through Lemon Squeezy. A license activates on up to two Macs at once; manage activations via Menu → Manage License….

As of 1.6, if the trial expires before you're ready to buy, the activation window offers a "Test for 5 minutes" button — two sessions per day, resetting at local midnight — so you can verify a fix against your gear without committing first.

§ 13

Updates & launch at login #

Lyra updates itself with Sparkle. You're notified in the menu when a new version is available; the release notes render right in the update window. Optionally enable Launch at login so your keys are ready the moment you sit down.
