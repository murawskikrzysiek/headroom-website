Wiring Lyra into every Mac automation surface

App Intents, Focus Filters, x-callback URLs. One change underneath, several different shapes on top.

Krzysiek Murawski 15 May 202610 min read

Lyra 1.5 is scriptable from every Mac automation surface I could find. Shortcuts, Spotlight, Focus Filters, Stream Deck, Keyboard Maestro, Raycast. One change underneath — every action is now an App Intent with a structured return — several different shapes on top.

This post is about the architecture of that, not the feature list. If you just want to know what to type into Spotlight, the user guide has it.

One surface, four entry points

Apple gave us four ways to expose a piece of app functionality to the rest of the system, and they each speak to a different part of a user's workflow.

App Intents are the underlying definition. You write one Swift type per action, declare its parameters and return type, and from that single source Apple builds you a Shortcuts action, a Spotlight result, a Focus Filter step, and a Siri-callable verb.

x-callback URLs sit alongside, not underneath, the Intents. They're the lowest common denominator — anything that can fire a URL on macOS can drive Lyra, and that includes Stream Deck, Keyboard Maestro, Raycast, AppleScript, even a hand-typed Terminal command.

The trick was making one change underneath that earned the four surfaces, not building four parallel systems.

Before 1.5, Lyra had the URL scheme but no App Intents. Adding the latter without rewriting the former meant treating the existing URL handlers as the canonical implementation and writing the Intents as a thin shim that called into them. Same code, two front doors.

x-callback returns, by example

Every endpoint accepts an optional x-success URL. When the action completes, Lyra appends the structured return as flat query parameters to that URL and opens it. No JSON parsing required in Shortcuts.

$ open "lyra://volume/get?x-success=shortcuts://x-callback-url/run-shortcut?name=Show+Level"

# Shortcut receives:
#   tapered = 0.58
#   db      = -10.2

The flat-query format means a Shortcut can read db directly from Shortcut Input → db with no parsing step. That's the kind of small affordance that decides whether someone actually builds the automation they had in mind.

Wrapping up

The takeaway, if there is one: it costs little to expose the same underlying actions through multiple Apple-native surfaces, once you commit to writing them as structured intents rather than as a bag of strings. Spotlight, Shortcuts, Focus, and Siri all come along for free. The URL scheme stays useful for the long tail.
